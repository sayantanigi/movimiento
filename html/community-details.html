Videoask Widget is overlaid with Videoask Widget how to dismiss first?

If you're seeing two overlapping VideoAsk widgets on your website or app, you might have multiple widgets embedded or some configuration issue causing them to stack up. Here's a step-by-step guide to help you dismiss the first and prevent this in the future...

  1. Check Your Code: Ensure that the code snippet for embedding the VideoAsk widget is not duplicated or improperly structured in your site's HTML or JavaScript files.

  2. Unique Widget ID: If you're embedding multiple VideoAsk widgets on the same page, each one should have a unique identifier or a unique configuration to distinguish them.

  3. Debug Mode: Enable browser developer tools (usually by pressing F12 or Ctrl+Shift+I on your keyboard) and inspect the HTML structure of the page. Look for the widget elements and identify whether they're duplicated or overlapping.

  4. Custom Close Button: Some widgets include a built-in close button. If you see a close (X) button on one of the widgets, you can manually dismiss it.

  5. Modify or Remove the First Widget:

    • If the widget is created dynamically via JavaScript, ensure only one instance is initialized.
    • Adjust or remove the widget that's intended to be dismissed. You can set the widget's CSS to display: none temporarily as a quick fix.

  6. Clear Cache: Sometimes, the issue might be due to caching. Try clearing your browser cache or disabling caching to see if this resolves the problem.

  7. Check for Custom Settings: If your widget configuration includes settings for delays or triggers, ensure that these aren't conflicting between the two widgets.

  8. Contact Support: If the issue persists, consider contacting VideoAsk support for further assistance.
